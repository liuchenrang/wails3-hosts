# 🔍 问题分析：hosts 文件没有内容

## 问题描述

用户报告：点击应用配置后，hosts 文件里没有写入开启组的 hosts 内容。

## 根本原因

通过诊断脚本发现：**所有分组都未启用** (`is_enabled: false`)

### 诊断结果

```
分组统计:
  总分组数: 2
  启用的分组: 0  ← 关键问题！

问题发现: 没有启用的分组！
```

虽然用户添加了 hosts 条目，但分组本身没有被启用，导致生成的内容为空。

## 工作原理

### hosts 内容生成的条件

代码中的逻辑 (`internal/domain/service/hosts_domain_service.go:33-37`):

```go
for _, group := range groups {
    entries := group.GetEnabledEntries()
    if len(entries) == 0 {
        continue  // ← 跳过空分组和未启用分组
    }
    // ...
}
```

**只有满足以下条件才会写入 hosts 文件**:
1. ✅ 分组存在
2. ✅ 分组已启用 (`group.IsEnabled = true`)
3. ✅ 分组中有启用的条目 (`entry.Enabled = true`)

### GetEnabledEntries 的逻辑

```go
func (g *HostsGroup) GetEnabledEntries() []HostsEntry {
    if !g.IsEnabled {
        return []HostsEntry{}  // ← 返回空数组
    }

    result := make([]HostsEntry, 0, len(g.Entries))
    for _, entry := range g.Entries {
        if entry.Enabled {
            result = append(result, entry)
        }
    }
    return result
}
```

**关键**: 如果 `group.IsEnabled = false`，直接返回空数组，不会有任何内容写入。

## 解决方案

### 方法 1: 在应用界面中启用分组

1. 打开 Hosts Manager 应用
2. 在左侧分组列表中，找到要启用的分组
3. **点击分组左侧的复选框** ✓
4. 确保复选框显示为选中状态
5. 点击"应用配置"按钮
6. 输入 sudo 密码

### 方法 2: 手动修改配置文件


## 常见问题

### Q: 为什么默认分组不启用？

A: 遵循**最小惊讶原则**和**安全原则**:
- 新创建的分组默认不启用
- 需要用户明确选择启用哪些分组
- 避免意外的配置写入系统文件

### Q: 条目的 enabled 和分组的 is_enabled 有什么区别？

A:
- **entry.Enabled**: 控制单个条目是否输出
- **group.IsEnabled**: 控制整个分组是否输出

**两者都必须为 true 才会输出**:
```
group.IsEnabled = true  AND  entry.Enabled = true  →  输出
group.IsEnabled = false AND  entry.Enabled = true  →  不输出
group.IsEnabled = true  AND  entry.Enabled = false →  不输出
```

### Q: 如何检查哪些分组已启用？

A:
1. 在应用中查看：分组左侧有 ✓ 标记的为已启用
2. 查看配置文件: `cat ~/Library/Application\ Support/hosts-manager/config.json | grep "is_enabled"`
3. 运行诊断脚本: `./scripts/diagnose-hosts-write.sh`
