## 🚀 快速开始

### 编译应用

```bash
# 开发模式
wails3 dev

# 生产模式
go build -o bin/hosts-manager .
```

### 基本使用流程

1. **创建分组**
   - 点击左侧 "+" 按钮
   - 输入分组名称和描述
   - 点击"确定"

2. **添加 hosts 条目**
   - 点击刚创建的分组
   - 在右侧编辑器中输入条目：
     ```
     127.0.0.1    example.local    # 这是注释
     192.168.1.1    test.server
     ```

3. **启用分组** ⚠️ 重要
   - 在左侧分组列表中
   - 点击分组左侧的 **☐ 复选框**
   - 确保显示为 **☑** 选中状态

4. **应用配置**
   - 点击顶部"应用配置"按钮
   - 输入 sudo 密码
   - 配置将写入 /etc/hosts

## ⚠️ 重要提示

### 为什么 hosts 文件是空的？

如果应用配置后 hosts 文件没有内容，最可能的原因是：**分组未启用**

**解决方法**:
1. 在左侧分组列表中找到您的分组
2. 点击分组左侧的复选框 ☐
3. 确保显示为 ☑（已选中）
4. 再次点击"应用配置"

### 如何检查分组是否启用？

**方法 1: 查看应用界面**
- 已启用的分组左侧有 ✓ 标记
- 未启用的分组左侧是空的 ☐

**方法 2: 运行诊断脚本**
```bash
./scripts/diagnose-hosts-write.sh
```

**方法 3: 查看配置文件**
```bash
cat ~/Library/Application\ Support/hosts-manager/config.json | grep "is_enabled"
```

## 🔐 安全说明

### 密码处理

- ✅ sudo 密码**不会**写入 hosts 文件
- ✅ 密码只在内存中保存，不持久化
- ✅ 密码缓存 5 分钟后自动清除
- ✅ 支持手动清除密码缓存

### 如果之前使用了有漏洞的版本

如果您之前使用过旧版本，**请立即检查**:

```bash
# 检查 hosts 文件是否包含泄露的密码
sudo grep -n '[a-zA-Z0-9]\{8,\}' /etc/hosts
```

如果发现密码泄露：
1. 备份 hosts 文件：`sudo cp /etc/hosts /etc/hosts.backup`
2. 编辑删除密码行：`sudo nano /etc/hosts`
3. 修改系统 sudo 密码

## 🛠️ 故障排除

### 问题 1: 应用配置后 hosts 文件为空

**原因**: 分组未启用

**解决**:
1. 在左侧分组列表中，点击分组左侧的复选框
2. 确保显示为 ✓ 选中状态
3. 再次应用配置

### 问题 2: 提示需要 sudo 密码

**这是正常的！** 写入 /etc/hosts 需要管理员权限。

**解决**:
- 输入当前用户的 sudo 密码
- 密码会被缓存 5 分钟
- 不需要每次都输入

### 问题 3: 应用配置失败

**检查**:
1. sudo 密码是否正确
2. 是否有写入 /etc/hosts 的权限
3. 磁盘空间是否充足
