```go
// ApplyHosts() 不再接收密码参数
func (h *HostsHandler) ApplyHosts() error
```

**修改位置**: `frontend/src/App.tsx:213-243`

```typescript
const handleConfirmApply = async () => {
  try {
    // 1. 先验证 sudo 密码
    const validationResult = await hostsApi.validateSudoPassword(sudoPassword)

    if (!validationResult.valid) {
      alert('sudo 密码验证失败: ' + validationResult.error)
      return
    }

    // 2. 检测冲突
    const detectedConflicts = await hostsApi.detectConflicts()
    if (Object.keys(detectedConflicts).length > 0) {
      setConflicts(detectedConflicts)
      setShowConflicts(true)
      return
    }

    // 3. 应用配置（不需要传递密码）
    await hostsApi.applyHosts()

    // 4. 清理状态
    setShowSudoPrompt(false)
    setSudoPassword('')
    await loadVersions()
    alert(t('common.success'))
  } catch (error) {
    console.error('Failed to apply hosts:', error)
    alert(t('common.error'))
  }
}
```

## 新的工作流程

```
用户输入密码并点击确认
    ↓
1. 调用 validateSudoPassword(password)
   → 验证密码
   → 提升 sudo 权限
   → 系统缓存凭证（5分钟）
    ↓
2. 验证成功
    ↓
3. 检测配置冲突
    ↓
4. 调用 applyHosts()
   → 无需密码参数
   → 使用系统缓存的 sudo 凭证
   → 写入 hosts 文件
    ↓
5. 完成
```

## 安全性保证

### ✅ 密码不会泄露的原因

1. **验证阶段**:
   - `sudo -v` 从 stdin 读取密码
   - 验证成功后，系统会缓存 sudo 凭证
   - 密码**不写入文件**

2. **应用阶段**:
   - `sudo sh -c "cat > /etc/hosts"`
   - 使用系统缓存的凭证
   - **不需要传递密码**
   - **密码不在 stdin 中**

3. **结果**:
   - ✅ 密码只在验证时通过 stdin 一次
   - ✅ 应用配置时 stdin 只有 hosts 内容
   - ✅ 密码不会写入 `/etc/hosts`

## 技术要点

### sudo -S 的工作原理

```bash
# 验证阶段
echo "password" | sudo -S -v
# sudo 从 stdin 读取密码
# 验证成功后，系统缓存凭证（5分钟）

# 应用阶段
sudo sh -c "cat > /etc/hosts" < hosts_content
# 使用系统缓存的凭证
# 不需要密码
```

### 为什么现在安全了？

1. **密码只在验证时使用**: `sudo -v` 验证密码
2. **应用时不传递密码**: `ApplyHosts()` 无密码参数
3. **使用系统缓存**: `sudo sh -c "cat > /etc/hosts"` 利用系统凭证
4. **内容纯净**: `cat` 命令的 stdin 只有 hosts 内容
